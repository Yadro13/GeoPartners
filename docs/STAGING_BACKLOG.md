# Backlog тестового середовища GeoPartners

Цей файл містить технічні та продуктові роботи, які можна виконувати у staging без додаткових рішень замовника. Питання, що потребують погодження, ведуться окремо у `CUSTOMER_QUESTIONS.md`.

## Правила просування

- Уся поточна розробка виконується у гілці `staging`.
- Railway staging автоматично розгортає гілку `staging`.
- Гілка `main` та Railway production не змінюються до погодженого релізу.
- Перенесення у `main` виконується після проходження перевірок відповідного батча у staging.
- Дані PostgreSQL і Bucket переносяться у production окремою контрольованою процедурою перед запуском.
